# 원료비 원인분석 (Streamlit)

월별 생산계획 · 원료 단가/실적을 받아, **원료비 변동을 단가 / 물량 / 믹스**로 분해하고
**어떤 제품·어떤 원료**가 움직였는지 추적하는 앱.

## 핵심 원리
```
원료비 = 이론사용량 × 단가
이론사용량 = 계획중량 × BOM배합률   (키블 반제품은 하위원료로 전개)
Δ원료사용량 = Σ제품( Δ생산중량 × BOM배합률 )   ← 사용량 변화의 제품 귀속
```

## 실행
```
pip install -r requirements.txt
streamlit run app.py
```

## 페이지
1. `app.py` — 월 비교: 총 원료비 워터폴(단가/총생산량/믹스), 제품·원료 기여, BOM 대비 실적 대사
2. 제품 드릴다운 — 생산량·원료비 추이, 원료 구성
3. 원료 드릴다운 — 단가·사용량 추이 + 사용량 변화의 제품 귀속
4. 단가 감시 — 단가 변동 원료 + 원료비 영향도 랭킹
5. 데이터 관리 — 월 파일 업로드·검증·DB 반영

## 데이터 (`data/`)
- `main_db.xlsx` — 시트: `원료단가사용`(년월·원료코드·단가·실적사용kg·실적금액), `생산계획`(년월·표준제품·계획중량)
- `bom_master.xlsx` — 시트: `BOM`, `원료마스터`
- `mapping.xlsx` — 시트: `매핑표`(파일 품목명 → 표준제품)

## 월 마감 (매달)
`데이터 관리` 페이지에서 ① 계획중량 파일 ② 단가·사용량 파일 업로드 → 검증 통과 시 DB append.
BOM/배합비가 바뀌면 `bom_master.xlsx` 갱신.
